## 2. The five files + contracts

### (a) `src/server/visitors/counter.ts` ‚Äî server module (`server-only`)
- Key **`visits:total:${ZUGZWANG_ENV}`** ‚Äî founder-pinned, non-negotiable, env-**last** (deliberately unlike `getRedisKey`'s env-first; documented in-file). Inline `VALID_ENVS` validation (fail-loud, mirrors `keys.ts` posture) ‚Üí different keyspace from rate-limit, so ordering divergence is intentional.
- `incrementAndRead(): Promise<number>` ‚Üí `redis.incr(key)` (coerced `Number(...)`).
- `read(): Promise<number>` ‚Üí `redis.get<string>(key)`; `null`‚Üí`0`; parse int, non-finite‚Üí`0`.
- Reuse the singleton `redis` (bounded transport). **Errors throw** (route maps them).

### (b) `src/app/api/visits/route.ts` ‚Äî POST only, `export const dynamic = 'force-dynamic'`
- Order: `isbot(user-agent)?` ‚Üí per-IP cap? ‚Üí `incrementAndRead()`.
- Bot **OR** cap exceeded ‚Üí return **current total via `read()`**, no increment, HTTP 200.
- Per-IP sliding window **60 / min**, constant **module-local** (`VISITS_PER_IP_PER_MIN`) ‚Äî NOT in `config/limits.ts`, NOT a ¬ß11 row. Own `Ratelimit` instance, prefix `getRedisKey("ratelimit","visits-ip")` (env-first ¬ß11 disjointness convention). Reuse `ipIdentifier` + local `extractIp`.
- Any Redis/counter throw (unreachable) ‚Üí `{ total: null }`, HTTP 200. **Never 429, never an error body, never 500.**
- **Never log IP or user-agent** (ADR-0007). No `logRequest`.
- Body shape: `{ total: number | null }` (founder-pinned; plain `Response.json`, not the error envelope).
