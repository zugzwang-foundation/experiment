**Decisions made — founder rulings (a)–(f) (ratified).**
- **(a) Market-state gate = WRAPPER-level, COARSE.** After the pool lock, a PLAIN intra-tx `SELECT markets.status` — **no row lock**, must NOT enter the lock-order spine (SERIALIZABLE/SSI catches a concurrent flip as 40001→retry; a `markets` row lock here would risk deadlock vs W-3's `markets→…` order). `Open` → run callback; non-Open → throw `MarketNotOpenError(status)` (a product error, NOT retried). ENGINE.8 maps status→§15 code (`Closed→market_closed_at`, `Resolving→in_flight_timeout` coarse reject-all, others→`market_not_open`). Fine F-BET-6 window DEFERRED (S1).
- **(b) `insertEvent` = CALLBACK calls it; wrapper events-UNAWARE.** The tracker's "appends events via ENGINE.6" is the callback's `insertEvent(tx,…)` inside the wrapper's tx (`events` terminal in the chain).
- **(c) CPMM compute = CALLBACK** (reads locked reserves → `computeBuy`/`computeSell` → writes). Wrapper generic.
- **(d) `I-ATOMICITY-001` lands in ENGINE.7, WRAPPER-level** (SPEC.2 §14.4). Canonical assertion = a representative callback writes `pools`+`positions`+`dharma_ledger` then throws **MID-SPINE** (before `comments`/`bets`/`events`) → assert **zero rows across ALL tables** (full-chain rollback). No `I-ATOMICITY-002`; ENGINE.8 full-flow atomicity is acceptance-level under `tests/server/bets/`.
- **(e) Concurrency-test split = 7/2.** ENGINE.7 owns 7 in `tests/server/bets/concurrency.test.ts` (serializable-isolation-enforced, pool-row-lock-acquired, no-key-update-allows-fk-share, canonical-lock-order [4-table], retry-on-40001, retry-on-40P01, retry-budget-exhausted-emits-alarm-3). The 2 handler-level (idempotency-replay-skips-moderation-and-txn, moderation-outside-transaction) → ENGINE.8.
- **(f) Events-idempotency = PROPERTY TEST, not a named invariant** (per §14.1 — no `I-EVENT-IDEMPOTENT-001`). `bets::concurrency-retry-events-idempotent`: force 40001 → exactly one `events` row + `created_at` stable across the retry. **Retry-purity contract recorded first-class:** the wrapper re-runs the WHOLE callback on retry, so callbacks MUST generate all IDs (`event_id`, `bet_id`, `comment_id`) **and `metadata.idempotency_key`** at handler-entry and close over them — never per-attempt — else `ON CONFLICT (event_id, created_at)` never dedupes.
